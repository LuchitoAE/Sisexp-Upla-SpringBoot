BLOQUE 1 — SPRING BOOT + SPRING CLOUD
═══════════════════════════════════════════

Título: "Stack Seleccionado para SISEXP"

Subtítulo: "Spring Boot 3.4.1 + Spring Cloud 2024"

Componentes del ecosistema:
┌─────────────────────────────────────────┐
│ Spring Cloud Gateway       — API Gateway │
│ Spring Cloud Netflix Eureka — Discovery │
│ Spring Cloud OpenFeign     — REST Client │
│ Spring Cloud Config        — Config Git  │
│ Spring Cloud CircuitBreaker — Resilience4j │
│ Spring Boot Starter Web    — REST APIs   │
│ Spring Boot Starter Data JPA — DB access │
│ Spring Boot Starter Security — JWT/Auth  │
│ Spring Boot Starter Actuator — Health    │
└─────────────────────────────────────────┘

Ventajas (ícono check verde):
✓ Ecosistema unificado: todas las dependencias son compatibles entre sí
✓ Auto-configuración: sin XML, sin boilerplate
✓ Soporte empresarial: respaldado por VMware/Tanzu
✓ Documentación extensa: guías oficiales + comunidad masiva
✓ Integración nativa con Docker: buildpacks, layered jars

Ámbito de aplicación:
- Adecuado para equipos Java, sistemas empresariales medianos-grandes
- Ideal cuando el equipo ya conoce Spring Boot (curva de aprendizaje baja)
- Recomendado para proyectos que requieren alta mantenibilidad a largo plazo

BLOQUE 2 — ALTERNATIVAS EVALUADAS
═══════════════════════════════════════════

Título: "Opciones Consideradas y Descartadas"

Alternativa 1 — Node.js + Express + Consul:
- Ventaja: mismo lenguaje que el frontend (JavaScript)
- Desventaja: inconsistencia con stack Java del curso, ecosistema fragmentado
  (Express no tiene equivalente directo a Feign, Eureka, Gateway integrados)
- Decisión: DESCARTADO

Alternativa 2 — Go + gRPC + Kubernetes:
- Ventaja: máximo rendimiento, tipado fuerte con protobuf
- Desventaja: curva de aprendizaje alta, plazo del proyecto insuficiente para 
  dominar Go + gRPC + K8s simultáneamente
- Decisión: DESCARTADO

Alternativa 3 — Quarkus (Supersonic Subatomic Java):
- Ventaja: tiempo de inicio rápido, menor consumo de memoria
- Desventaja: ecosistema menos maduro que Spring Boot, menos documentación en 
  español, menor adopción en la industria peruana
- Decisión: DESCARTADO (viable para futuro)

═══════════════════════════════════════════
BLOQUE 4 — CONTRIBUCIÓN A REQUISITOS ABP
═══════════════════════════════════════════

Título: "Cómo el Framework Elegido Satisface los Requisitos del Proyecto"

Tabla de trazabilidad Framework → Requisito:

| Requisito ABP | Cómo lo resuelve Spring Cloud |
|---|---|
| Escalabilidad | Servicios independientes, scale out horizontal |
| Mantenibilidad | Cambios aislados por servicio, sin redeploy total |
| Seguridad (CIA) | JWT centralizado en Gateway, cada servicio valida token |
| Interoperabilidad | REST APIs estándar, JSON como formato único |
| Disponibilidad | Circuit Breaker evita fallos en cascada |
| Trazabilidad | Spring Cloud Sleuth + Zipkin (IDs de traza entre servicios) |
| Configuración externalizada | Spring Cloud Config desde Git |
| Despliegue continuo | Docker + Railway (o Render) con CI/CD desde GitHub |
