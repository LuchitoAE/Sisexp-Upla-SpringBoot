Patrones integrados en esta implementación:
✓ API Gateway — punto único de entrada, rate limiting, auth centralizada
✓ Database per Service — cada servicio dueño de sus datos
✓ Saga (Coreografía) — transacciones distribuidas con RabbitMQ
✓ Event-Driven — notificaciones desacopladas vía mensajería
✓ Circuit Breaker — fallos aislados, sin efecto cascada

Principios SOLID en esta arquitectura:
✓ SRP — cada microservicio tiene UNA responsabilidad (auth, presupuesto, etc.)
✓ OCP — nuevo comportamiento añadiendo servicios, sin modificar existentes
✓ LSP — cada servicio implementa contratos REST intercambiables
✓ ISP — endpoints específicos por cliente (nadie consume lo que no necesita)
✓ DIP — depende de abstracciones (RabbitMQ, Feign interfaces), no de implementaciones
